# Day-38-YAML-Basics

YAML is a simple, human-readable format used for configuration in DevOps tools like Docker and CI/CD. It uses key-value pairs (`key: value`) and is very strict about indentation. Only spaces are allowed—tabs will cause errors. Lists can be written using dashes (`- item`) or inline with brackets (`[item1, item2]`). YAML also supports nested data using proper spacing. It’s widely used because it is clean, easy to read, and ideal for configuration files.
